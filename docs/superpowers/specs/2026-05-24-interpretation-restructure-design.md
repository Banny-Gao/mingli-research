# 知识库内容规范重构设计

> 日期：2026-05-24
> 状态：Draft
> 范围：SPEC-interpretation.md 重构

---

## 一、背景

当前 SPEC-interpretation.md 采用强制七段式结构（入门导读→原文精读→义理解析→命造拆解→图解→常见误解→总结），适合教学场景但不适合所有篇目，且与产品定位中的"研究者"需求不完全匹配。

---

## 二、设计原则

### 2.1 内容驱动形式

解读的结构由内容本身决定，不是由模板强制规定。

**核心原则：**
- 不强制任何固定章节数量
- 不强制章节顺序
- 根据原文复杂度、理论密度、命造数量自适应结构

### 2.2 专业定位

目标读者是有积累的专业人士，不需要降低门槛或过度简化。

**原则：**
- 不强行通俗化，保持专业术语准确性
- 密度高时使用总分总结构
- 允许长篇幅，不限制字数

### 2.3 底线约束

所有解读必须严格遵守 `SKILL-bazi-research-dispute.md`：

- **经典原文为唯一准绳**
- **实事求是客观中立**
- **流派分歧客观呈现，不武断判定**
- **经典引用必须标注来源**

---

## 三、解读结构模式

根据内容类型，推荐以下结构模式（推荐非强制）：

### 3.1 短篇模式（原文简短、理论集中）

适用：原注+任氏曰各一段，无命造或只有1个命造

```
## 核心论点
### 论点名称
【原注】原文
→ 义理解读

【任氏曰】原文
→ 义理解读

（可选）命造验证

## 常见误解
错误说法
→ 经典原文反驳
```

### 3.2 标准模式（理论+命造）

适用：有多层次理论、有2-4个命造

```
## 总论
一句话概括本篇核心

## 理论解析
### 论点一
原文依据 + 义理解读

### 论点二
原文依据 + 义理解读

## 命造验证
【命造一】乾造：xxx
格局分析 → 结论

【命造二】...
...

## 关联篇章
（可引用其他篇目的理论关联）
```

### 3.3 密集模式（内容丰富、层次多）

适用：理论复杂、命造密集（5个以上）

```
## 总论
本篇核心 + 在体系中的位置

## 理论一
### 原文解读
### 义理展开
### 命造验证

## 理论二
### 原文解读
### 义理展开
### 命造验证

## 理论三
...

## 总结
核心要点 + 进阶路径
```

## 四、质量要求

### 4.1 原文引用规范

- 每次引用必须标注来源：`【原注】` 或 `【任氏曰】`
- 必须引用完整句子，不截取半句
- 引用后必须有解读，不引不解

### 4.2 命造规范

- 所有命造必须有经典出处，禁止自创
- 优先使用任铁樵原注中的命造
- 禁止使用网络命造或民间流传命造

### 4.3 术语处理

- 首次出现时加注：`**术语**：解释
- 不强制所有术语都加注（专业读者可自行理解基础术语）

### 4.4 图解使用

| 使用场景 | 禁用场景 |
|---------|---------|
| 逻辑链条清晰的因果关系 | 文字已说清楚的简单概念 |
| 三元/三方对比关系 | 装饰性图示 |
| 流程判断类 | 牺牲准确性换简洁 |
| 五行生克循环 | 强行简化复杂理论 |

---

## 五、规范底线清单

以下为不可突破的底线：

1. **必须引用原文** — 解读必须有【原注】或【任氏曰】原文依据
2. **必须解读** — 禁止只堆砌原文不解读
3. **禁止自创命造** — 所有命造必须有经典出处
4. **禁止网络伪论** — 不得传授无经典依据的理论
5. **流派分歧处理** — 分列各方依据，不武断判定唯一正确
6. **引用规范** — 必须标注经典全称+篇章+原文

---

## 六、与 generate.js 的关系

**结论：不需要适配**

generate.js 对 interpretation 结构完全无感知：
- 读取 markdown → 转 HTML → 存起来
- 搜索索引：stripHtml 提取纯文本

interpretation 结构变化不影响 generate.js 输出。

**如果有前端展示需求**，可按需定义 Markdown 渲染规范，但当前 MVP 不需要。

## 七、产出规范

### 7.1 自检清单

输出前逐项检查：

- [ ] 读取 source.md 确认原文已合规录入
- [ ] 解读结构符合内容复杂度（不过度模板化）
- [ ] 所有原文引用标注来源
- [ ] 所有命造有经典出处
- [ ] 术语首次出现已加注
- [ ] 图解用于逻辑展示，非装饰
- [ ] 无触碰底线清单

### 7.2 规范优先级

| 层级 | 内容 |
|------|------|
| **强制** | SKILL-bazi-research-dispute.md + 底线清单 |
| **推荐** | 结构模式建议、总分总（密集内容时） |
| **可选** | 术语加注、图解使用 |
