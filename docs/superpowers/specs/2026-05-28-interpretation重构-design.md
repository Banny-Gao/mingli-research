## 一、核心定位

**双受众**：专业研究者（深度、引申、恍然大悟）+ 普通大众（一知半解但能跟上）

**实现方式**：每章节增加「通俗注释层」【白话：...】，专业内容后附白话解读。

## 四、关键设计详解

### 4.1 复杂度评分制

LLM 自行评估三个维度，综合判定模式：

| 维度 | 说明 | 分值 |
|------|------|------|
| 理论密度 | 独立理论点的数量和深度 | 1-5 |
| 案例数量 | 原典自带命造/占例数量 | 1-5 |
| 跨篇章关联度 | 与本书其他篇章的理论关联程度 | 1-3 |

**总分 = 理论密度×2 + 案例数量 + 跨篇章关联度**

| 分数 | 模式 | 建议骨架 |
|------|------|----------|
| ≤ 5 | 短篇 | 精简结构，核心论点为主 |
| 6-10 | 标准 | 总论 + 理论解析 + 案例验证 |
| ≥ 11 | 密集 | 多理论模块 + 配套案例 + 总结 |

### 4.2 通俗注释层

每个论点/段落后，LLM 按需添加【白话】通俗注释：

```markdown
## 论点一

> 【原文】完整原文

表层义理解读……

【白话】一句话概括本段核心含义，用比喻或日常语言让普通读者理解。

升华洞见……

【白话】升华部分的通俗解读。
```

### 4.3 升华洞见扩大边界

**允许范围扩大**：
- 本篇在全书体系中的定位、作用
- 与本书其他篇章的理论关联
- 原文理论在正统实践中的应用指导
- 学界公认的补充考据、观点汇总
- **新增**：深层引申含义、跨领域类比启示、历史背景分析

**绝对禁止**（不变）：
- 野诀、私门理论、网络伪论
- 武断流派定论
- 脱离原文的个人臆想

### 4.4 推荐骨架示例

**短篇模式**（总分 ≤ 5）
```markdown
## 核心论点

### 论点一
> 【原文】
义理解读
【白话】

### 论点二（如有）
> 【原文】
义理解读
【白话】

### 篇章定位（升华洞见）
【白话】
```

**标准模式**（总分 6-10）
```markdown
## 总论

本篇核心主旨

## 理论解析

### 论点一
> 【原文】
义理解读
【白话】

## 案例验证

【命造/占例一】
分析 + 【白话】

## 篇章关联（升华洞见）
【白话】
```

**密集模式**（总分 ≥ 11）
```markdown
## 总论

本篇核心主旨 + 全书体系定位

## 理论一

### 原文解读
> 【原文】
【白话】

### 义理展开
【白话】

### 配套案例

## 理论二（依次）

## 总结（升华洞见）
【白话】
```

### 4.5 禁止行为风格（沿用 general.md）

```markdown
## 四、禁止行为【红线】

### 4.1 通用禁止【触碰即违规·零容忍】

1. 严禁脱离经典原文自创理论、无据臆断
2. 严禁断章取义、节选曲解经典
3. 严禁贬低、否定正统经典及公认固定名家的合规言论
4. 严禁传播网络伪论、杂论野诀、民间无据口诀
5. 严禁对流派争议武断判定唯一正确，强行偏袒
6. 严禁堆砌经典原文只引不解，以典代论

### 4.2 LLM专属禁止

7. 严禁擅自修改、篡改、修正原文字句、异文
8. 严禁脑补补全原文残缺、脱漏内容
9. 严禁超出升华洞见允许范围自由发散
10. 严禁随意增删推荐骨架的章节结构
11. 严禁使用规范以外的 Markdown 语法
```

## 五、执行流水线（不变）

1. 前置文件校验
2. 内容复杂度评分 → 模式判定
3. 梳理内容结构
4. 逐段引用 + 表层解读 + 【白话】
5. 案例/分歧处理
6. 升华洞见（含【白话】）
7. 图解补充（按需）
8. 自动化自检
9. 输出最终文件

## 六、自检清单（调整）

### 🔴 致命错误
- [ ] 引用是否截取半句？
- [ ] 是否出现自创案例？
- [ ] 是否脱离原文编造观点？
- [ ] 升华洞见是否出现野诀/自创？
- [ ] 【白话】是否曲解原义？

### 🟠 格式错误
- [ ] 注家标注格式统一？
- [ ] 引文使用 `>` 块引用？
- [ ] 结构与模式匹配？
- [ ] 无装饰性图表？

### 🟡 内容检查
- [ ] 表层解读完整？
- [ ] 升华洞见深度足够？
- [ ] 【白话】通俗但不错误？
