## 一、目标

按 SPEC-catalog 的「卷→篇」二级目录范式重整 `呱呱集` 结构：

- 上集「名人命造」按人物拆分（一段一人物）
- 下集「命运答客问」按题材归类（一组问答一主题）

下集无内部分组，按韦千里自列题材「婚姻 财气 儿女 事业 寿元 行运 流年 月建」归类。落盘时按 `(答XXX君)` 起始位置划界。

**归类边界问题需拆分阶段单列人工 gate：**
- "答林严再娶" → 婚姻（虽然也涉及子嗣）
- "答胡锡珍" → 子嗣（虽然问子女也问婚姻）
- "答曾耀森" 出现两次（一次答本造，一次答令郎） → 第一次归命学综论/事业，第二次归子嗣
- 长篇相法答（如答朱曙人中、答孟焕如达摩相书）→ 相法杂问

## 九、风险与人工 gate

| 风险 | 缓解 |
|---|---|
| 上集段段横跨多人（如 24/25/26 篇是组合人物） | 已合并为「组合人物篇」单篇；如读者反馈粒度粗可再拆 |
| 下集同一问跨多主题（如答曾耀森涉及子嗣+命运综论） | 按主问归类（哪一问最长归哪一类）；边界争议项列 gate 人工裁定 |
| 文件夹名中文特殊字符 | 命名用纯中文，不用空格/标点；与现有千里命稿一致 |
| 旧文件夹删除导致 git 误删 | 用 `git mv` 而非 rm；分批 commit |

## 十、执行顺序

1. 创建新 articles 目录骨架（35 个空文件夹）
2. 按映射表拆分 source.md（先上集后下集）
3. 重写 catalog.md
4. 校验：行数合计、# 标题、章节编号
5. 删除旧 3 个粗粒度文件夹
6. 运行 generate.js（不在本次执行，仅记录后续）
7. git add + commit（按章节分批，避免单 commit 过大）
