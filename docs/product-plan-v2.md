## 一、产品愿景与定位

**愿景：** 成为华人命理学研究者首选的学术级古籍阅读平台，以严谨的注解体系、可执行的AI技能驱动差异化于所有现有竞品。

**定位：** 中医古籍/古典文献研究平台路线，专业深度远超豆瓣阅读式古典内容聚合，真正做到"读得懂、用得上、能验证"。

**核心差异化：**
- 现有竞品（百度贴吧命理版块、古籍网、民间八字论坛）：仅提供原文，无注解，无技能，无法执行
- 本平台：原文 + 逐层学术注解 + 可执行的命理推理技能 → 学者能用，研究者能验证

---

## 二、用户画像与核心场景

| 用户类型 | 特征 | 核心需求 |
|---|---|---|
| **命理自学者** | 25-45岁，有古文基础但缺系统路径 | 找到入门路径、逐篇学习、验证理解 |
| **从业者** | 执业命理师，需要快速查阅原典依据 | 快速定位篇目、一键调取技能执行 |
| **学术研究者** | 中文系/历史系/哲学系硕博 | 引用标准、引文规范、多文本对照 |
| **AI研究者** | 构建命理推理系统 | 标准化的技能文件、SKILL.md规范接口 |

**核心旅程：** Landing页选典籍 → BookApp读篇目 → 打开Modal细读注解 → 学习配套技能 → 记录笔记书签 → 产出自己的解读

## 三、现状评估

### 已构建（Strong Foundation）
- 双页架构：Landing（选典籍）+ BookApp（读篇目）
- 动态内容加载：per-chapter 懒加载 + 同步编译
- 进度系统：localStorage 读写状态 + 书签系统
- 阅读体验：TOC侧边栏 + 阅读进度条 + 回到顶部
- 全局搜索：`/` 快捷键，跨篇目跨技能
- 数据管道：generate.js 自动从 catalog.md 驱动，catalog 即真相源
- 暗色主题：完整 Tailwind CSS 主题系统
- 移动端适配：700px / 480px 两档响应式

### 关键缺口

| 维度 | 现状 | 目标 |
|---|---|---|
| 内容覆盖 | 滴天髓 9/64 篇（14%） | 64/64 + 渊海子平 |
| 搜索深度 | 仅标题匹配 | 全文+注解内搜索+高亮 |
| 笔记系统 | 仅书签（布尔型） | 划线批注+私有笔记 |
| 技能整合 | 技能独立展示 | 技能与篇目双向关联 |
| 导出打印 | 无 | PDF/打印友好样式 |
| SEO | 有基础 Helmet | 结构化数据+落地页 |
| 数据持久化 | 仅 localStorage | 云端备份（可选） |
| 可访问性 | 字体可调，无ARIA | WCAG 2.1 AA |

## 四、产品路线图

### P0 — Must Have（启动前必须完成）

#### P0.1 全文搜索引擎
**描述：** 在 SearchBar 基础上建立全文索引，搜索篇目注解内容而非仅标题。支持高亮跳转。

**用户价值：** 研究者无需记住篇名，可搜"三元""旺衰""格局"等关键词直接定位。

**复杂度：** 低

#### P0.2 进度仪表盘强化
**描述：** Landing页的全局进度统计扩展：当前在读篇目、最近阅读、连续学习天数（localStorage）。

**用户价值：** 强化"学习进度"感知，提升回访率。

**复杂度：** 低

#### P0.3 深化滴天髓解读（持续）
**描述：** 优先完成上篇剩余5篇（09-14），再推进下篇。按 PROGRESS.md 分批推进。

**复杂度：** 内容生产，非技术

---

### P1 — Should Have（下一迭代）

#### P1.1 技能与篇目双向关联
**描述：** 在篇目详情页显示相关技能卡片，在技能详情页显示相关篇目。

**复杂度：** 中

#### P1.2 划线+批注系统
**描述：** 在 Modal 阅读模式下，支持选中文字划线（存储于 localStorage，带颜色：重点/疑问/引用），支持添加私有批注。批注列表在侧边栏展开。

**复杂度：** 中

#### P1.3 读书笔记页
**描述：** 新增 `/notes` 路由，聚合所有划线批注，按典籍/篇目分组，支持导出为 markdown。

**复杂度：** 低-中

#### P1.4 渊海子平接入
**描述：** 参照滴天髓的 `books/ditiansui-site/` 结构，创建 `books/yuanhaiziping/` 目录和 catalog.md。

**复杂度：** 中（内容生产为主）
