# 模式 B：文本

## 输入契约

- **必填：** 纯文本内容
- **可选位置：**
  - 用户直接粘到聊天
  - `.txt` / `.md` 文件绝对路径
  - 多文件批量

## 主 agent 动作

1. 收源（接收文本 / `Read` 文件）
2. LLM 清洗：
   - 去除多余空行（保留段间单空行）
   - 识别 `【XX】` 注家标记，转为 `> 【XX】` 块引用
   - 保留原文一字不改
3. 写 source.md：H1 + 正文 + 注家块

## 清洗原则（来自 SPEC-source.md §五）

LLM 清洗时**必须**遵守：

1. 严禁混入解读内容
2. 严禁修改原文用字
3. 严禁添加现代标点以外的标记（仅 `> 【注家名】` 块引用是例外）
4. 严禁分段处理长段原文
5. 严禁添加空行以外的任何内容

## 失败兜底

| 异常 | 处置 |
|------|------|
| 路径不存在 / 文件不可读 | 报错退出，让用户重给 |
| 文本为空 | 报错退出 |
| 文本 < 100 字 | 警告并继续（短文也可能是完整章节）|
| 注家标记格式混乱 | 询问用户手动标注 |

## 与模式 A 的差异

- 模式 A：HTML 解析 + 提取
- 模式 B：直接清洗（无 HTML 解析）
- 两者都遵守 SPEC-source.md 红线
