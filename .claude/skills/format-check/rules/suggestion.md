# 🔵 建议规则（R8-R13）

大部分仅交互模式提示。R10（连续空行）、R13（行尾空白）在 `--fix` 下自动修复，其余规则不自动修。

---

## R8：可用表格未用

**检测逻辑：**
1. 扫描连续 3 行以上含 `|` 或 `、` 分隔的对比性数据行
2. 行内容包含对照结构（如"命造X：... 命造Y：..."、"前者...后者..."）
3. 当前未使用 GFM 表格格式
4. 判定为可用表格优化

**fix_type:** `manual`

**交互模板：**
```
检测到对比性内容（第 L1-L2 行），当前用纯文本排列，可转为 GFM 表格提升可读性。
[转为表格] [跳过]
```

---

## R9：缺少分节线

**检测逻辑：**
1. 检测 `## ` 二级标题之间的内容区块 > 50 行
2. 两个大区块之间无 `---` 分隔
3. 判定为缺少视觉分节

**fix_type:** `manual`

**交互模板：**
```
检测到大章节之间缺少分节线（第 L 行前后），建议添加 --- 增强视觉分隔。
[添加 ---] [跳过]
```

---

## R10：连续空行冗余

**检测逻辑：**
1. 扫描连续 3 个以上的空行
2. 判定为空行冗余

**fix_type:** `auto`

**修复策略（`--fix` 自动执行）：**
- 将 3+ 连续空行缩减为 2 个空行

**交互模板：**
```
检测到 N 处连续空行冗余（3+ 空行），建议规范为 1-2 行。
[自动规范] [跳过]
```

---

## R11：中英文间缺空格

**检测逻辑：**
1. 扫描中文汉字后紧跟英文单词（`[一-鿿][A-Za-z]`）
2. 或英文单词后紧跟中文汉字（`[A-Za-z][一-鿿]`）
3. 判定为中英文间缺空格

**fix_type:** `manual`

**交互模板：**
```
检测到 N 处中英文间缺空格（如"甲木日主"与"react-markdown"混排），建议添加空格改善阅读体验。
[自动添加空格] [跳过]
```

---

## R12：强调标记不一致

**检测逻辑：**
1. 扫描全文中的关键术语（如"用神"、"格局"、"日主"、"食伤"等命理术语）
2. 统计每个术语的出现次数和加粗次数
3. 术语出现 3 次以上但仅部分加粗 → 标记不一致
4. 术语首次出现未加粗 → 建议首次加粗

**fix_type:** `manual`

**交互模板：**
```
检测到强调标记不一致：
- "用神" 出现 8 次，仅 3 次加粗
- "格局" 出现 5 次，首次出现未加粗
建议：统一加粗策略（首次出现 + 关键论断处）。
[统一加粗] [仅首次加粗] [跳过]
```

---

## R13：行尾空白

**检测逻辑：**
1. 扫描行尾存在空格或 Tab 字符的行
2. 判定为行尾空白（git diff 噪音）

**fix_type:** `auto`

**交互模板：**
```
检测到 N 处行尾空白，会在 git diff 中产生噪音。
[自动清除] [跳过]
```
