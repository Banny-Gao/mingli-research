## 流程记录（结构性推理）

- **Step 1 模式：** 锁定单点
- **Step 2 源模式：** 锁定 A URL
- **Step 3 收源 + 抓取：**
  - WebFetch URL
  - HTML 复杂（正文与广告/导航/页脚混杂）→ 用户在 Step 3 主动给 CSS selector
  - LLM 用 selector 定位主区域（如 `article.main` / `#content` / `.article-body`）
  - 在该 DOM 节点内提取原文（不混广告/页脚）
- **Step 4 字形策略 gate：** 视 catalog.md 声明
- **Step 5 落盘：** 4 选项

## L2 关键差异（vs L1）

| 维度 | L1 | L2 |
|------|----|----|
| 用户输入 | URL | URL + CSS selector |
| 提取精度 | 依赖 LLM 自行判断 | selector 辅助精准定位 |
| 适用 | 简洁静态页 | 复杂 HTML（混广告）|
| 失败回退 | 无 | selector 不匹配 → 回退 L1 |

## 限制

L2 模式需要用户主动给 selector，依赖用户对目标页面 HTML 结构的了解。常见 selector 模式：
- `article` / `main` / `.content` / `#content` / `.article-body`
- 维基百科：`#mw-content-text`
- 百度百科：`.main-content` 或 `#bodyContent`
