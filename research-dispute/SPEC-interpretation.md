# 解读规范

> **适配场景：** 大模型自动化批量生成 `books/**/articles/{篇名}/interpretation.md` 解读文件
> **上游依赖：** `SPEC-source.md`（原文必须提前录入并校验）
> **下游产出：** `SPEC-skill.md`（可选）
> **领域硬约束：** 必须严格遵守 `research-dispute/general.md` 及对应术数类别专项文件（如 `bazi.md`）
> **前置自检义务：** LLM 必须在进入内容生成阶段前**完整通读**上述两个文件并输出明确确认（详见 §五 Step 1），仅阅读本规范 §1.5 内联摘要**不构成**对正本的替代

---

## 一、设计原则【核心】

### 1.1 内容驱动形式

不强制固定章节数量、顺序。**LLM 按内容灵活组织，三种模式（短篇/标准/密集）仅作结构参考，模式选择不强制。**

#### 模式判据（可计算口径，强制）

```python
def mode_of(text: str) -> str:
    n = sum(len(line) for line in text.splitlines() if line.strip())
    return "短篇" if n < 500 else "标准" if n < 2000 else "密集"
```

统计口径：原文+原注所有非空行字符数（含标点、含夹注，不含标题与分隔符 `---`）

#### 基础定义（统一统计口径）

- **有效原文字符**：仅统计正文、注解文字，不含空行、分隔符
- **独立理论点**：原文中独立成义、可单独解读的观点/法则
- **有效案例**：原典/原注中附带的命造、占例、实例，**仅统计经典原生案例**

#### 极短篇精简原则（强制·短篇模式专用）

`mode_of()` 返回"短篇"（< 500 字符）时，原文信息量不足以支撑长篇展开。LLM 必须遵守：

- **行文与原文体量匹配**：不强行拉长
- **核心任务收束**：说清"原文在说什么" + "在全书中的位置"
- **宁可留白，不增字解经**

> 注：本节与 §1.2 §1「短篇原文文本分析」互补——前者管"别硬撑篇幅"，后者管"原文本体必须做透"。两节不冲突，可同时生效。

#### 反模板规则（强制）

§三 篇章组织原则仅作指引，默认按内容自由组织。

- **标题反机械化规则（强制）**：禁止将 source.md 的排版层次机械翻译为二级标题——不得出现 source 分层标签作为标题。标题应简洁，从内容中提炼，不做定义式展开、不用冒号分句、不做概念解释。
- **同批互检**：若同批任意两篇存在完全相同的二级标题，需审视标题同质化缺陷，判断是否需要必须重新设计。

### 1.2 专业定位与解读深度

内容由浅入深展开——从逐句训诂自然过渡到方法论提炼，不切硬边界。浅层释义照顾普通大众（一知半解但能跟上），深层引申照顾专业研究者。二者在写作中为一体而非分属不同章节。通俗注释融入写作语言（详见 §1.2 §3）。

#### 1. 表层义理（必写）

内容范围：
- 逐段还原**原文本义**
- 完整转述各注家公开注解、主流释义
- 结合案例做字面+格局/流程解读
- 引用任铁樵疏解时段名取自然标题，不必机械标注

约束：**所有内容必须有 `SPEC-source.md` 原文/注文作为依据**。

**注家分层要求（强制）**：原文存在多层注解时（如【原注】+ 后世注家），LLM 必须：
- 先逐层转述各注家的独立贡献，再注明注家之间是否存在实质性的义理推进（如原注平实、后世注引入新的哲学框架或生成论体系），不做混同处理
- 此要求在 `mode_of()` 返回"短篇"时同样适用——不因篇幅短而跳过注家分层

**短篇原文文本分析（强制）**：`mode_of()` 返回"短篇"时，深化洞见可免（见下），但原文**本体**的文本分析不可跳过。LLM 必须覆盖：
- 原文**每一句**的句式与结构（如条件句、对仗、设问等——其句式本身往往内含方法论主张）
- 关键**用字**的隐喻义场与义理负载（如"宗"为何不用"本""源""根"；"先观"为何不是"先知""先识"）
- **篇名**的选址意义（为何全书以此篇开篇、篇名有何统摄性）

注疏可以简，原文本体不可一笔带过。

#### 2. 深化洞见

> 本节**非必写**——是否需要撰写深化洞见，由 LLM 根据原文内容**自行判断**：原文确有深层意蕴可深入时撰，原文信息单薄或表层已尽意时不强写，可融入表层义理末段一笔带过。**篇幅长短不作为是否撰写的判据**——短篇亦可有深层意蕴，长篇亦可表层已尽。

> ⚠️ **文件输出禁词**：LLM 不得在 interpretation.md 中出现"升华洞见"四字——本节内容属性为深化洞见，但不以此四字显式标注。

✅ **允许范围**

1. 本篇在全书整体典籍体系中的定位、作用
2. 本书体系内的方法论定位（笼统表述，**不做具体篇章呼应断言**）
3. 原文理论在正统实践中的客观应用指导
4. 学界/正统注家公认的补充考据、观点汇总
5. 深层引申含义、跨领域类比启示、历史背景分析

❌ **绝对禁止**

1. 自创口诀、私门理论、网络野诀、民间偏方
2. 主观判定流派高下、武断定义"唯一正确观点"
3. 脱离原文的个人臆想、玄学延伸、自由发挥
4. 编造未见于原典的观点、解读、案例
5. 无前置跨篇读取依据时的具体跨篇断言（如"本篇与第 X 章呼应""任氏在 Y 章亦有此论"）——如要做跨篇关联，必须能在本篇原文中找到直接证据

#### 3. 通俗注释层【内嵌于表层义理】

**不要单独起一行【白话】。** 通俗注释应直接融入表层义理的写作语言中，让 LLM 用"读者听得懂"的方式写解读。

**必须通俗化（融入写作语言，不可堆砌术语）的硬场景：**

1. **命造/占例的格局判定**（如"此造身旺用财""此例变爻用事"）—— 判定结论后用一句话说清"意味着什么"
2. **流派分歧的客观陈列**（多派观点并列时）—— 介绍完两派观点后，用一句话点明分歧焦点
3. **原文晦涩或含生僻字**（无现代标点的骈文、典故用典）—— 翻译成现代汉语，不要直接放过

**可保留学术语言的场景：**

1. 原文引用块（必须用原句）
2. 跨篇章关联的术语（保持学术一致性）
3. 注家原文的转述（保留术语原貌）

### 1.3 SPEC 自身语言一致性

本规范要求 LLM 输出"通俗、融入写作语言"（§1.2 §3），但 SPEC 正文使用一定学术术语（如"形上／形下""体用""生成论"等）作为**元语言**以精确界定要求。LLM 执行时须严格区分：

- **SPEC 作为元规范**：可使用学术术语精确描述约束条件
- **interpretation.md 作为产出**：面向普通读者，不得镜像 SPEC 的元语言语域

禁止将 SPEC 自身的措辞风格直接复制到解读文件中。解读的语言门槛应始终低于本规范的语言门槛。

---

### 1.4 底线约束【不可突破，LLM 高优先级执行】

1. **经典原文为唯一准绳**：所有观点必须标注原典出处，引用完整原句，禁止截取半句
2. **实事求是客观中立**：不盲从门派、不刻意偏向某一方观点
3. **流派分歧处理**：分列各方原文依据，只做客观陈列，不判定对错
4. **引用规范标注**：完整标注经典全称+篇章+注家，格式统一
5. **LLM 附加约束**：不得擅自修改、修正、补全原文异文/脱漏，仅做客观标注
6. **学术语体规范（反 AI 痕迹）**：解读文本须呈现为传统学术注疏的语体风格，**禁止**使用以下揭示 LLM 身份或流水线痕迹的元自我引用与技术术语——
   - **元自我引用**：禁止"本解读""本文""本篇解读"等自称（"本篇"指原文篇章时除外）；元表态如"本解读不据此改动原文字面""本解读亦不强行补全""本解读按 SPEC §X.X 处理"等
   - **元自我标签**：禁止"【原文此处疑似 OCR 错字】""【本解读不展开具体跨篇断言】"等带【】的元自我标签；OCR 异文/存疑字直接以学术注释形式表述（如「X」疑为「Y」之误），不使用方括号元标签
   - **流水线技术术语**：禁止 `mode_of()`、`SPEC §X.X`、`按 SPEC 公式判为`、`按用户口径`、`按标准档组织` 等仅对生成流水线有意义、对读者无意义的内部术语
   - **元数据块**：禁止在文首插入"**本篇模式**""**模式判定**""**体量定位**""**mode_of**"等元数据 blockquote 段；如需说明原文体量，以"本篇条文短小"等学术叙述代之
   - **改写方向**：以传统训诂学、校勘学、命理注疏的语体为模板——「此言……」「按……」「盖……」「观此造……」等表述方式自然，避免任何"我作为 AI 解读了这段"的语感
7. **流派分歧与跨篇引述的元表态边界**：当因 §1.2 §2 ❌项 5 等红线需克制具体跨篇/跨书引述时，**不得用"本解读不就此作前置跨书引述"等元表态方式呈现**；以"原文未明指……具体何书""后世研究各有发挥"等客观陈述方式代之

### 1.5 通用红线禁令【general.md §2 内联摘要】

> 完整版本见 `general.md §二、绝对红线禁令`（14 条），此处为 LLM 必读核心：

1. 严禁脱离经典原文自创理论、无据臆断；禁用非正统体系规则
2. 严禁断章取义、节选曲解经典；严禁以名家注解替代经典原文做最高判准
3. 严禁贬低、否定正统经典及**公认固定名家**的合规正统言论
4. 严禁传播网络伪论、杂论野诀、民间无据口诀、私自编造体系
5. 严禁对有经典支撑的流派争议下"唯一正确"绝对定论
6. 严禁面对自身错误质疑拒不认错、回避辩解
7. 严禁偏离核心议题无关发散、过度延展
8. 严禁堆砌经典原文只引不解、以典代论
9. 严禁学术交流中人身攻击、嘲讽贬低、傲慢抬杠
10. 严禁用"仅供参考、个人观点"规避规则责任
11. 严禁被指出引用错误后拒不认错、回避抵赖
12. 严禁篡改经典原文、拼接不同经典冲突内容
13. 严禁引用非公认名家、网络零散资料作为核心立论依据
14. 严禁把正统流派合理义理分歧强行判定为错误

**术数类别专项约束**：八字解读需同时遵守 `bazi.md`（如适用）。

## 二、质量规范

前端渲染依赖 `react-markdown + rehypeRaw`，仅允许使用指定 Markdown 语法。

允许语法：块引用 `>`、二级~四级标题 `##`/`###`/`####`、**粗体**、表格、`mermaid` 代码块、原生简单 HTML。

### 2.1 原文引用规范

#### 通用格式模板

```markdown
> 【注家标识】完整原文整句/完整段落（不截断、不删减）

义理解读正文（引后必解，无纯引文；术语当场释义，融入写作语言，不另起【白话】行）
```

#### 注家标识规则

1. 优先读取 `catalog.md` 预设注家名称，严格沿用
2. **兜底规则**（`catalog.md` 缺失时强制使用）：
   - 原典籍正文：【原文】
   - 原书自带注解：【原注】
   - 多位注家集体评述：【诸家评】
   - 后世学者补充注解：【后人补注】
   - 文中夹注小字：【夹注】

#### 引文溯源精确性（强制）

注家（如任铁樵）用语常熔铸多部经典（如《周易》《尚书》《中庸》《诗经》等），LLM 在为注家措辞标注经典出处时：

- **多源则多举**：若注家用语可追溯到多部经典，须并举，不得单一指归
- **不确定则缓冲**：无法精确定位时使用缓冲表述（如"语近《XX》XX 篇""与《XX》义理同类""熔铸先秦宇宙论诸说"），不做斩钉截铁的单源断言
- **原文用典与注家用典分开**：原文自身用典（如《滴天髓》正文引《系辞》）与注家注文用典（如任氏引《尚书》《系辞》）须区分标注，不混同

### 2.2 命造/案例规范

1. 所有案例**仅限原典、原注自带内容**，LLM 禁止自创、改编、增补
2. 案例必须标注来源：`【命造一（原注第X段）】`/`【占例二（原文附例）】`
3. 解读顺序：罗列案例基础信息 → 格局/流程分析 → 结合原文得出结论

### 2.3 术语处理

术语解释融入写作语言（详见 §1.2 §3），不另起格式行。行业通用术语无需加注；小众/篇章特有术语单篇内首次出现须解释，重复出现不再标注。

### 2.4 图解使用规则

#### ✅ 必须使用图解的场景

1. 存在多级逻辑判断、先后推演流程、五行/阴阳循环、格局推导
2. 多流派观点横向对比、多个案例并列对比
3. 典籍体系脉络、理论源流、篇章关联梳理
   > **短篇模式不触发**——篇章体系可在文字中笼统说明，不强制使用 Mermaid 图解。

#### ❌ 禁止使用图解的场景

1. 纯文字释义、单一句子解读、无逻辑分支的简单论述
2. 装饰性配图、无实际信息的图形
3. 能用文字+表格清晰说明，额外增加流程图

## 三、篇章组织原则

不预设固定章节模板。组织时遵循两条线索：

1. **按原文的独立理论点切分章节**。每个二级标题对应原文中一个可独立成义的观点或论证段落，标题从原文关键词中抓取（反机械化规则见 §1.1）。章节数量由原文的论点数量决定，与原文的字符数无关。
2. **案例跟随其所属的理论点**。原文在哪个论点下举了例，案例就放在哪个章节里融入解读，不单独抽离为独立章节。原文无案例则无案例章节。

结尾须点明篇章在全书的笼统定位（不做具体跨篇断言），可融入末节或独立成节。

### 跨门类命名

- 八字/命理：案例标注「命造」
- 卜筮/六爻：案例标注「占例」
- 相学/堪舆：不设命造模块
- 古医书：不设命造类格式

## 四、禁止行为【红线】

### 4.1 通用禁止

内容生成的通用红线与 `general.md` §2 一致，此处不重复。LLM 执行解读生成时须同时遵守 `general.md` 全部 14 条红线禁令。

### 4.2 解读生成专属禁止

1. 严禁擅自修改、篡改、修正原文字句、异文
2. 严禁脑补补全原文残缺、脱漏内容
3. 严禁超出深化洞见允许范围自由发散
4. 严禁随意增删篇章组织原则规定的必需内容
5. 严禁使用规范以外的 Markdown 语法、自定义复杂样式

## 五、LLM 执行流水线

大模型按以下步骤执行。**Steps 1-2 必须在内容生成前完成且不可跳步**。Steps 3-7 首次通过须走完整流程，后续允许回跳迭代（如写到 Step 6 时发现结构需调整，可回 Step 3 重新梳理；在 Step 4 解读过程中自然产出深层洞见时，无需等到 Step 6 才写入）。

1. **前置自检（Prerequisite Self-Inspection）** — LLM 在进入内容生成前必须依次完成以下三项自检，全部通过后方可进入 Step 2：
   - 1.1 **文件存在性自检** — 确认 `SPEC-source.md`、`catalog.md`、`research-dispute/general.md` 及对应术数类别专项文件（如 `bazi.md`）均存在且可读；任一缺失立即终止并输出缺失清单
   - 1.2 **正本通读自检** — 完整通读 `research-dispute/general.md` 全文（含 `§一` 总则与 `§二` 红线 14 条）以及本规范 §1.5 内联摘要；输出明确确认：`✅ 已完整通读 general.md 全文 + §1.5 内联摘要`
   - 1.3 **专项约束通读自检** — 完整通读对应术数类别专项文件（如八字读 `bazi.md`、六爻读 `liuyao.md`）；输出明确确认：`✅ 已完整通读 {术数} 专项约束文件`
2. **红线确认** — 完整通读 `research-dispute/general.md §二、绝对红线禁令` 14 条；逐条对照本规范 §1.5 内联摘要核对一致性；输出明确确认：`✅ 已完整通读 general.md §二 14 条红线 + §1.5 内联摘要，且 14 条与摘要完全一致`
3. **内容结构梳理** — 拆分原文段落、标记注家、分类理论与案例；从篇章独特论点中提取标题（反机械化规则见 §1.1），禁止用 source 分层标签作标题
4. **逐段引用+表层解读** — 套用引文模板，「一引一解」；术语当场释义，融入写作语言（§1.2 §3）
5. **案例/分歧处理** — 有案例则按模板解读；存在流派分歧则客观陈列
6. **深化洞见（按需）** — LLM 根据原文内容自行判断是否撰写：原文确有深层意蕴时撰，无则不强写；术语当场释义融入语言
7. **图解补充（按需）** — 对照触发规则，满足条件则添加 Mermaid 图表
8. **自评合规分** — LLM 按"产出自检清单"逐项自评，给出 0-5 分合规分；低于 4 分强制重写
9. **输出最终文件** — 自评通过后，输出完整 `interpretation.md`

## 六、古籍异常场景处理

### 1. 版本异文

```
> 【原文】主流版本完整语句
> 【异文标注】另一版本作：XXX

解读：客观罗列异文差异，不评判版本优劣。
```

### 2. 字句脱漏/原文残缺

固定标注：`【原文此处残缺/字句脱漏】`，仅基于现有完整内容解读。

### 3. 原文无任何注解

仅解读【原文】本义，不强行杜撰注解。

### 4. 原文无任何案例

不设案例章节，其余内容保留。

### 5. 内容重复、传抄冗余

正常解读，可在解读中注明「本段为传抄重复内容」。

### 6. 超长原文（单篇字符 > 5000）

不拆分文件，在原有章节内增加四级小节 `####` 分段梳理。

## 七、产出自检清单

### 🔴 致命错误（阻断输出，强制修正）

- [ ] 引用是否截取半句、删减原文？
- [ ] 是否出现自创案例、改编案例？
- [ ] 是否脱离原文、编造无依据观点？
- [ ] 是否武断判定流派对错、偏袒某一方？
- [ ] 是否擅自修改、补全残缺/异文原文？
- [ ] 深化洞见是否出现自创理论、野诀？
- [ ] 是否触碰 `general.md §二` 14 条红线中的任何一条？
- [ ] 是否触碰 `SPEC-interpretation.md §1.5` 内联摘要中的任何一条？

### 🟠 格式错误（强制修正）

- [ ] 注家标注格式是否统一？缺失 catalog.md 时是否使用兜底标识？
- [ ] 引文是否全部使用 `>` 块引用？
- [ ] 通俗注释是否按 §1.2 §3 强制场景融入表层义理写作语言？是否错误地另起【白话】行？
- [ ] 图解是否符合「必用/禁用」规则？
- [ ] 是否使用了规范以外的 Markdown 语法、自定义复杂样式？

### 🟡 内容检查（优化调整，不阻断输出）

- [ ] 表层解读是否完整覆盖原文含义？
- [ ] 深化洞见是否紧扣全书体系、无冗余内容？
- [ ] 通俗注释是否通俗但不曲解原义？
- [ ] 内容是否由浅入深自然展开、无突兀跳跃？

### 📊 自评合规分（强制 · 内部过程）

LLM 在输出前必须按上述清单逐项自评，给出 **0-5 分合规分**——此分作为 LLM 内部决策依据（决定是否重写），**不写入 interpretation.md 文件**。

- **5 分**：致命错误 0 项 + 格式错误 0 项 + 内容检查 0 项
- **4 分**：致命错误 0 项 + 格式错误 0 项 + 内容检查 ≤ 1 项
- **3 分及以下**：存在致命错误，或格式错误 ≥ 1 项，或内容检查 ≥ 2 项 → **强制重写**

**低于 4 分的输出视为不合规产物，禁止提交。**
