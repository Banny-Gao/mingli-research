# 解读规范

> **适配场景：** 大模型自动化批量生成 `books/**/articles/{篇名}/interpretation.md` 解读文件
> **上游依赖：** `SPEC-source.md`（原文必须提前录入并校验）
> **下游产出：** `SPEC-skill.md`（可选）
> **领域硬约束：** 必须严格遵守 `research-dispute/general.md` 及对应术数类别专项文件（如 `bazi.md`）

---

## 一、设计原则【核心】

### 1.1 内容驱动形式（复杂度评分制）

不强制固定章节数量、顺序。**LLM 自行评估内容复杂度，综合判定所属模式。**

#### 复杂度评分维度

| 维度 | 说明 | 分值 |
|------|------|------|
| 理论密度 | 独立理论点的数量和深度 | 1-5 |
| 案例数量 | 原典自带命造/占例数量 | 1-5 |
| 跨篇章关联度 | 与本书其他篇章的理论关联程度 | 1-3 |

**总分 = 理论密度×2 + 案例数量 + 跨篇章关联度**

| 分数 | 模式 | 建议结构 |
|------|------|----------|
| ≤ 5 | 短篇 | 精简结构，核心论点为主 |
| 6-10 | 标准 | 总论 + 理论解析 + 案例验证 |
| ≥ 11 | 密集 | 多理论模块 + 配套案例 + 总结 |

#### 基础定义（统一统计口径）

- **有效原文字符**：仅统计正文、注解文字，不含空行、分隔符
- **独立理论点**：原文中独立成义、可单独解读的观点/法则
- **有效案例**：原典/原注中附带的命造、占例、实例，**仅统计经典原生案例**

---

### 1.2 专业定位与解读深度

**双受众**：专业研究者（深度、引申）+ 普通大众（一知半解但能跟上）

解读必须分层输出**表层义理 + 升华洞见 + 通俗注释**三层内容。

#### 1. 表层义理（必写）

内容范围：
- 逐段还原**原文本义**
- 完整转述各注家公开注解、主流释义
- 结合案例做字面+格局/流程解读

约束：**所有内容必须有 `SPEC-source.md` 原文/注文作为依据**。

#### 2. 升华洞见（必写）

✅ **允许范围**

1. 本篇在全书整体典籍体系中的定位、作用
2. 与本书其他篇章的理论关联、前后呼应关系
3. 原文理论在正统实践中的客观应用指导
4. 学界/正统注家公认的补充考据、观点汇总
5. 深层引申含义、跨领域类比启示、历史背景分析

❌ **绝对禁止**

1. 自创口诀、私门理论、网络野诀、民间偏方
2. 主观判定流派高下、武断定义"唯一正确观点"
3. 脱离原文的个人臆想、玄学延伸、自由发挥
4. 编造未见于原典的观点、解读、案例

#### 3. 通俗注释层【白话】（必写）

每个论点/段落后，LLM 按需添加【白话】通俗注释，供普通读者理解。

标注格式：
```markdown
【白话】一句话概括本段核心含义，用比喻或日常语言。
```

---

### 1.3 底线约束【不可突破，LLM 高优先级执行】

1. **经典原文为唯一准绳**：所有观点必须标注原典出处，引用完整原句，禁止截取半句
2. **实事求是客观中立**：不盲从门派、不刻意偏向某一方观点
3. **流派分歧处理**：分列各方原文依据，只做客观陈列，不判定对错
4. **引用规范标注**：完整标注经典全称+篇章+注家，格式统一
5. **LLM 附加约束**：不得擅自修改、修正、补全原文异文/脱漏，仅做客观标注

## 二、质量规范

前端渲染依赖 `react-markdown + rehypeRaw`，仅允许使用指定 Markdown 语法。

允许语法：块引用 `>`、二级~四级标题 `##`/`###`/`####`、**粗体**、表格、`mermaid` 代码块、原生简单 HTML。

### 2.1 原文引用规范

#### 通用格式模板

```markdown
> 【注家标识】完整原文整句/完整段落（不截断、不删减）

义理解读正文（引后必解，无纯引文）
【白话】通俗注释
```

#### 注家标识规则

1. 优先读取 `catalog.md` 预设注家名称，严格沿用
2. **兜底规则**（`catalog.md` 缺失时强制使用）：
   - 原典籍正文：【原文】
   - 原书自带注解：【原注】
   - 多位注家集体评述：【诸家评】
   - 后世学者补充注解：【后人补注】
   - 文中夹注小字：【夹注】

### 2.2 命造/案例规范

1. 所有案例**仅限原典、原注自带内容**，LLM 禁止自创、改编、增补
2. 案例必须标注来源：`【命造一（原注第X段）】`/`【占例二（原文附例）】`
3. 解读顺序：罗列案例基础信息 → 格局/流程分析 → 结合原文得出结论

### 2.3 术语处理

术语加注格式：`**术语**：简短释义`

- 行业通用术语无需加注
- 小众/篇章特有术语单篇内首次出现须加注，重复出现不再标注
- 释义控制在 15 字以内，不展开长篇解释

### 2.4 图解使用规则

#### ✅ 必须使用图解的场景

1. 存在多级逻辑判断、先后推演流程、五行/阴阳循环、格局推导
2. 多流派观点横向对比、多个案例并列对比
3. 典籍体系脉络、理论源流、篇章关联梳理

#### ❌ 禁止使用图解的场景

1. 纯文字释义、单一句子解读、无逻辑分支的简单论述
2. 装饰性配图、无实际信息的图形
3. 能用文字+表格清晰说明，额外增加流程图

## 三、推荐骨架（三种模式示例）

三大模式**章节层级仅作参考**，LLM 按内容灵活组织，正文内容自由创作。

### 3.1 短篇模式（总分 ≤ 5）

```markdown
## 核心论点

### 论点一

> 【注家标识】完整原文

表层义理解读……

【白话】通俗注释……

### 论点二（如有）

> 【注家标识】完整原文

表层义理解读……

【白话】通俗注释……

### 篇章定位（升华洞见）

本篇在全书体系中的作用、延伸思考。

【白话】通俗注释……
```

### 3.2 标准模式（总分 6-10）

```markdown
## 总论

本篇核心主旨。

## 理论解析

### 论点一

> 【注家标识】完整原文

表层义理解读……

【白话】通俗注释……

### 论点二（依次罗列所有理论点）

> 【注家标识】完整原文

表层义理解读……

【白话】通俗注释……

## 命造验证 / 占例解析

【命造/占例一（来源标注）】
案例信息 + 分析解读
【白话】通俗注释……

【命造/占例二（来源标注）】
案例信息 + 分析解读
【白话】通俗注释……

## 篇章关联（升华洞见）

理论串联、实践指导、全书体系定位。

【白话】通俗注释……
```

### 3.3 密集模式（总分 ≥ 11）

```markdown
## 总论

本篇核心主旨 + 全书体系定位。

【白话】通俗注释……

## 理论一

### 原文解读

> 【注家标识】完整原文

【白话】通俗注释……

### 义理展开

深度解析理论内涵。

【白话】通俗注释……

### 配套案例

【命造/占例一】
案例信息 + 分析解读

## 理论二（依次罗列所有独立理论模块）

### 原文解读

### 义理展开

### 配套案例

## 总结（升华洞见）

全篇核心要点汇总、跨篇章关联、正统实践指引。

【白话】通俗注释……
```

#### 跨门类适配说明

- 八字/命理：「案例验证」统一命名为「命造验证」
- 卜筮/六爻：「案例验证」统一命名为「占例解析」
- 相学/堪舆：删除命造模块，改为「图文解读/实例印证」
- 古医书：改为「方药/脉理解析」，禁用命造类格式

## 四、禁止行为【红线】

### 4.1 通用禁止

内容生成的通用红线与 `general.md` §2 一致，此处不重复。LLM 执行解读生成时须同时遵守 `general.md` 全部 14 条红线禁令。

### 4.2 解读生成专属禁止

1. 严禁擅自修改、篡改、修正原文字句、异文
2. 严禁脑补补全原文残缺、脱漏内容
3. 严禁超出升华洞见允许范围自由发散
4. 严禁随意增删推荐骨架的章节结构
5. 严禁使用规范以外的 Markdown 语法、自定义复杂样式

---

## 五、LLM 执行流水线

大模型**严格按以下步骤依次执行**，禁止跳步、颠倒顺序、遗漏步骤：

1. **前置文件校验** — 读取 `SPEC-source.md`、`catalog.md`、`general.md` 及对应术数类别专项文件（如 `bazi.md`）；若缺失，终止并输出提示
2. **复杂度评分** — 评估理论密度、案例数量、跨篇章关联度，计算总分，判定模式
3. **梳理内容结构** — 拆分原文段落、标记注家、分类理论与案例
4. **逐段引用+表层解读** — 套用引文模板，「一引一解」，添加【白话】
5. **案例/分歧处理** — 有案例则按模板解读；存在流派分歧则客观陈列
6. **撰写升华洞见** — 在允许范围内创作，添加【白话】
7. **图解补充（按需）** — 对照触发规则，满足条件则添加 Mermaid 图表
8. **自动化自检** — 运行「产出自检清单」，发现问题立即修正
9. **输出最终文件** — 自检全部通过后，输出完整 `interpretation.md`

## 六、古籍异常场景处理

### 1. 版本异文

```
> 【原文】主流版本完整语句
> 【异文标注】另一版本作：XXX

解读：客观罗列异文差异，不评判版本优劣。
```

### 2. 字句脱漏/原文残缺

固定标注：`【原文此处残缺/字句脱漏】`，仅基于现有完整内容解读。

### 3. 原文无任何注解

仅解读【原文】本义，不强行杜撰注解。

### 4. 原文无任何案例

删除骨架中「命造验证/占例解析」章节，其余章节保留。

### 5. 内容重复、传抄冗余

正常解读，可在解读中注明「本段为传抄重复内容」。

### 6. 超长原文（单篇字符 > 5000）

不拆分文件，在原有骨架内增加四级小节 `####` 分段梳理。

## 七、产出自检清单

### 🔴 致命错误（阻断输出，强制修正）

- [ ] 引用是否截取半句、删减原文？
- [ ] 是否出现自创案例、改编案例？
- [ ] 是否脱离原文、编造无依据观点？
- [ ] 是否武断判定流派对错、偏袒某一方？
- [ ] 是否擅自修改、补全残缺/异文原文？
- [ ] 升华洞见是否出现自创理论、野诀？

### 🟠 格式错误（强制修正）

- [ ] 注家标注格式是否统一？缺失 catalog.md 时是否使用兜底标识？
- [ ] 引文是否全部使用 `>` 块引用？
- [ ] 【白话】注释是否按需添加？
- [ ] 图解是否符合「必用/禁用」规则？

### 🟡 内容检查（优化调整，不阻断输出）

- [ ] 表层解读是否完整覆盖原文含义？
- [ ] 升华洞见是否紧扣全书体系、无冗余内容？
- [ ] 【白话】是否通俗但不曲解原义？
