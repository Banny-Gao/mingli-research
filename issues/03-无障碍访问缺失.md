# 补充问题二：可访问性（A11y）

> 生成时间：2026-05-17
> 优先级：🟢 低（但不可长期忽视）
> 关联：目标用户群体特征

---

## 现状

站点完全没有考虑 WCAG 标准。

| 维度                 | 状态   |
| -------------------- | ------ |
| 颜色对比度           | 未测   |
| 字体大小             | 未适配 |
| skip-to-content 导航 | 无     |
| 图片/视频 alt 文本   | 无规范 |
| 键盘导航             | 未测   |
| ARIA 标签            | 未使用 |
| 屏幕阅读器兼容性     | 未测   |

---

## 背景

国学内容受众有一半以上是 35+ 群体，视力下降、鼠标操作不熟练是现实需求。

可访问性不只是盲人友好，是**所有边缘用户群体**的友好度。

---

## 建议方案

**不现在做，但需要建立规范：**

1. 颜色对比度不低于 WCAG AA 标准（4.5:1）
2. 所有图片必须填写 alt 文本
3. 添加键盘可聚焦元素（button / a 必须有焦点样式）
4. 为 landmark 区域加 ARIA role
5. 设置基础 skip-to-content 链接

**执行方式：** 在 ESLint 规则中引入 `eslint-plugin-jsx-a11y`，强制检查 ARIA 规则。
