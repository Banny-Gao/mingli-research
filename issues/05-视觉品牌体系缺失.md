# 补充问题四：视觉品牌体系缺失

> 生成时间：2026-05-17
> 优先级：🟡 中
> 关联：五大板块扩展、小程序视觉统一

---

## 现状

当前有 `tailwind.config.js` 和 `theme.less`，但视觉规范分散：

- 颜色/字号/间距/圆角散落在组件样式文件中
- 没有统一的 design token 定义
- 各板块（未来山/医/相/卜）没有主题色切换机制

---

## 问题场景

**场景一：未来加新板块**
山/医/相/卜各有自己的主题色。当前没有设计 token，改配色就是全站搜索替换，漏一块就是 bug。

**场景二：小程序复用视觉**
排盘页面、命盘展示页需要和主站视觉一致。当前没有统一 token，小程序配色只能靠 copy，一改全改。

**场景三：暗色模式扩展**
当前有暗色/亮色切换，但 dark mode 的颜色散落在各个组件里，没有统一规范。后续想精细化暗色体验（不同灰度、不同对比度），没有依据。

---

## 建议方案

建立 design token 分层：

```
src/tokens/
├── color.ts          ← 基础色板（主色/辅助色/中性色/语义色）
├── typography.ts     ← 字号/行高/字重
├── spacing.ts        ← 间距系统（4px 基准网格）
├── radius.ts         ← 圆角
├── shadow.ts         ← 阴影层级
└── motion.ts        ← 动画时长/缓动曲线
```

组件不再直接写颜色值，引用 token：

```tsx
// ❌ 不再这样写
<div className="text-gray-800 bg-white">

// ✅ 改用语义 token
<div className="text-content-primary bg-surface-default">
```

**好处：**

- 换主题色只需改 token 文件
- 小程序/其他端可以复用同一套 token
- 暗色模式只需重新定义语义 token 的值
