## 一、产品定位

**《滴天髓》学术知识库**：面向命理学研究者的古籍原文+注疏解读+可执行技能的数字阅读平台。

> 关键词：研究者、古籍、数字阅读平台。

---

## 二、目标用户分层与真实需求

| 用户                 | 真实需求                           | 当前满足程度       |
| -------------------- | ---------------------------------- | ------------------ |
| 命理爱好者（初学者） | 想学滴天髓，但看不懂原文           | 基本满足           |
| 命理师（从业者）     | 查阅案例、验证理论                 | 部分满足           |
| **命理学研究者**     | **引用原文、追溯版本、做学术论证** | **部分满足**       |
| AI/开发者            | 调用技能做命理工具                 | 有框架但无实质内容 |

### 核心需求满足情况（更新）

1. **原文溯源**：出处、版本异文、不同注家的分歧 → ❌ 版本异文仍缺失
2. **交叉索引**：篇与篇之间的理论关联 → ⚠️ 有静态映射，无可视化
3. **快速定位**：一段话能否直接引用、精确位置在哪 → ⚠️ 有定位但无引用ID
4. **版本对比**：任铁樵注 vs 徐乐吾注 vs 梁湘润注的差异 → ❌ 无版本对比

## 五、技术架构问题

### 5.1 搜索系统 ✅ 已实现

| 指标 | 现状 |
|------|------|
| 搜索范围 | 章节解读、技能、原文全文 |
| 搜索算法 | Fuse.js 模糊搜索 + 精确匹配优先 |
| 关键词高亮 | ✅ 已实现 |
| 键盘导航 | ✅ 已实现（/ 打开，↑↓ 选择，Enter 确认） |

**缺失**：
- 篇内局部搜索（当前只能搜篇目级别）
- 搜索历史
- 热门搜索推荐

### 5.2 数据模型 ⚠️ 可扩展

当前 `ChapterInfo` 支持 `num, name, isDone, category`，搜索索引包含 `slug, title, type, key, text`。

**扩展建议**：
- 增加 `difficulty`（难度等级）
- 增加 `tags[]`（标签系统）
- source 增加 `version` 字段支持多版本

### 5.3 关联图谱 ⚠️ 静态映射

`assoc.ts` 包含 `interpToSkill` / `skillToInterp` 双向映射，但：
- 只是数据映射，非可视化组件
- 无前端交互式关系图
