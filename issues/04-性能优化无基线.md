## 当前可预见的性能风险

| 风险点 | 说明 |
|--------|------|
| 内容全部内联 TS | bundle 膨胀，TS 编译慢 |
| 无代码分割 | 64篇内容打包成一个大 JS |
| 字体全量加载 | `@fontsource-variable/geist` 未做条件加载 |
| 无图片懒加载 | 长列表页面会一次性加载所有图片 |
| 无资源压缩 | 生产构建未配置 gzip/brotli |

---

## 建议方案

**第一步（本周）：测基线**
```bash
# 安装 Lighthouse CI
npx lighthouse https://your-site.com --output=json --output-path=./lighthouse-report.json
```

记录当前各项指标，建立基准线。

**第二步（优化项，按优先级）：**
1. 生产构建启用 gzip/brotli 压缩
2. 为路由添加懒加载（React.lazy + Suspense）
3. 图片懒加载（Intersection Observer）
4. 字体子集化（只加载用到的字形）
5. 迁移内容到 md 文件后，bundle 会显著减小

**不需要现在就做，但需要先有数据。**
