## 问题一：内容与代码耦合（优先级：🔴 高）

**现状：**
64篇滴天髓的源码、解读、skill 数据全部内联进 `.ts` 文件。
- `src/data/ditiansui-site/source/*.ts` — 64个文件，每篇约200-500行中文字符串
- `src/data/ditiansui-site/interp/*.ts` — 64个文件，每篇约400-800行中文字符串
- `src/data/ditiansui-site/skill/*.ts` — 73个文件

**影响：**
- 每次修改内容都需要 build 才能预览（开发体验差）
- Git diff 全是中文字符串，无法追溯历史修改
- 非技术人员无法贡献内容
- 内容越多，TS 编译越慢
- 代码仓库里混杂了内容资产和代码资产

**建议方案：**
```
public/books/ditiansui/source/天道.md
public/books/ditiansui/interp/天道.md
public/books/ditiansui/skill/天道.md
```
代码只存数据结构定义和引用，内容用 Markdown 文件管理。build 时脚本注入。
优势：内容改动热更新无需 build，Git diff 可读，编辑门槛低。

**待确认：**
- [ ] 是否接受一次性迁移工作
- [ ] 迁移后内容管理流程（谁来维护、怎么更新）

## 问题二：H5 响应式未验证（优先级：🟡 中）

**现状：**
- Tailwind 有响应式断点，但当前没有明确写过 mobile-first 样式
- 没有移动端 hamburger menu / drawer 组件
- 没有在任何真机或模拟器上验证过

**影响：**
- 抖音引流来的用户 100% 是手机用户
- 如果 H5 体验差，用户直接流失
- 当前无法判断实际体验是否达标

**需要明确 H5 的定位：**

| 级别 | 说明 | 改造量 |
|------|------|--------|
| L1 轻量落地页 | 标题 + 一段引导语 + 跳小程序按钮 | 小 |
| L2 基础阅读 | L1 + 文章列表 + 单篇阅读（无笔记/搜索） | 中 |
| L3 完整体验 | L2 + 导航/搜索/笔记/收藏全功能移动端适配 | 大 |

**待确认：**
- [ ] H5 目标定位是 L1 / L2 / L3 中的哪一级？
- [ ] 是否接受阶段性改造（先 L1，后续迭代）？

## 问题三：页面路由过于扁平（优先级：🟡 中）

**现状：**
```
/              Landing
/notes         Notes
/:slug         BookApp（所有 64 篇 + 未来所有板块共用一个组件）
```

`BookApp` 一个组件承载了所有内容类型的渲染逻辑，内部靠 slug 判断走不同分支。

**影响：**
- 未来山/医/相/卜 四个板块各有不同的交互模式（不同格式的内容、不同的标注体系）
- 五六类内容逻辑全挤在一个组件里，代码膨胀，后续难以维护
- 每个板块独立 SEO 优化困难（meta 标签、页面结构）

**建议方案：**
```
/                          Landing（网站首页）
/books                     古籍知识库总览
  /books/ditiansui          滴天髓专区
    /books/ditiansui/tiandao  具体篇章
  /books/yuanhaiziping      渊海子平专区
/notes                     笔记系统
/tools                     工具集（未来）
```

**待确认：**
- [ ] 是否接受路由重构（影响 URL，可能影响 SEO）
- [ ] 各板块是否有不同的页面交互需求（目前先按相同处理）

## 问题四：缺少 CI/CD 和环境区分（优先级：🟢 低）

**现状：**
- 没有 dev/staging/prod 环境区分
- 没有自动化构建发布流程
- 没有移动端验证流程
- 构建依赖本地机器

**影响：**
- 多人协作困难
- 部署靠手动操作，有出错风险
- 移动端体验没有标准化的验证流程

**建议方案：**
```
GitHub Actions:
  - PR → 自动运行 TypeScript 编译 + Vitest + E2E
  - main merge → 自动构建 + 部署到 Vercel/Netlify
  - 移动端验证：在 CI 中用 Playwright 做 mobile viewport 测试
```

**待确认：**
- [ ] 是否有协作需求（多人编辑）？
- [ ] 部署目标是 Vercel / Netlify / 其他？
