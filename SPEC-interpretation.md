# 解读规范 v2.0

> **用途：** 规范 `books/**/articles/{篇名}/interpretation.md` 解读文件的产出
> **版本：** 2.0 — 重构自 v1.0 强制七段式，改为内容驱动形式
> **上游 SPEC：** SPEC-source.md（需先完成原文录入）
> **下游 SPEC：** SPEC-skill.md
> **硬约束（命理类）：** 必须严格遵守 `SKILL-bazi-research-dispute.md`；其他术数类别应遵守各自领域的学术研究规范

---

## 一、文件命名规范

```
articles/{篇名}/interpretation.md
```

- 篇名：中文篇名，与 catalog.md 中的篇名严格一致
- 每篇文章只有一个 interpretation.md 文件

---

## 二、设计原则【核心】

### 2.1 内容驱动形式

解读的结构由内容本身决定，不是由模板强制规定。

**强制要求：**
- 不强制任何固定章节数量
- 不强制章节顺序
- 不强制使用任何预设模板

**推荐模式（按内容复杂度选择）：**

| 模式 | 适用场景 | 结构特点 |
|------|---------|---------|
| 短篇模式 | 原文简短、无/少案例 | 直接论点，无需总分 |
| 标准模式 | 理论+2-4案例 | 总论→理论→案例→关联 |
| 密集模式 | 内容丰富、5+案例 | 总论→各理论独立展开→总结 |

### 2.2 专业定位

目标读者是有积累的专业人士。

**要求：**
- 保持专业术语准确性，不需要强行通俗化
- 内容密集时使用总分总结构
- 允许长篇幅，不限制字数

**解读深度：** AI 应在完全理解原文后，输出两层内容：
1. **表层义理** — 原文说了什么，注家如何理解
2. **升华洞见** — 该篇在全书体系中的位置、与其他篇章的理论关联、对实践的指导意义、可延伸的深层思考

不可止步于字面翻译，也不可脱离原文空谈升华。

### 2.3 底线约束【不可突破】

**命理类典籍**须严格遵守 `SKILL-bazi-research-dispute.md` 中的核心原则，适用于所有术数类别的通用底线：

1. **经典原文为唯一准绳** — 所有观点必须有经典原文支撑，标注出处（注家标记格式由各书 catalog.md 的元信息定义）
2. **实事求是客观中立** — 不盲从、不迎合、不偏袒门派
3. **流派分歧处理** — 分列各方依据，不武断判定唯一正确
4. **引用规范标注** — 经典全称+篇章+原文，不截取半句

## 三、质量规范

前端使用 react-markdown + rehypeRaw 渲染，interpretation.md 可使用以下 markdown 特性：

- `>` 块引用（引文分层）、`##`~`####` 标题层级、`**粗体**`、表格
- Mermaid 代码块（` ```mermaid `）用于流程图、时序图
- 原生 HTML（rehypeRaw 透传），用于复杂布局

### 3.1 原文引用

- 每次引用必须标注注家来源（如 `【原注】`、`【任氏曰】`，具体标记以各书 source.md 中使用的注家名为准）
- 必须引用完整句子，不截取半句
- 引用后必须有解读，不引不解
- 引文使用 `>` 块引用包裹，与解读正文分层

```
> 【注家名】原文完整句子

义理解读...
```

### 3.2 命造/案例规范（命理类）

- 所有案例必须有经典出处，**禁止自创**
- 优先使用原注中的命造
- 标注来源：注家曰第X段、命造一、命造二...

### 3.3 术语处理

- 首次出现时加注：`**术语**：解释`
- 专业读者可理解基础术语，不强制全部加注

### 3.4 图解使用

**必须使用：** 逻辑链条清晰、流程判断、体系结构、对比分析等该术数领域需要可视化的场景（如命理的三元对比和五行循环、医学的经络走向和脏腑关系、相术的面部分区）

**禁止使用：** 文字已说清的概念、装饰性图示、牺牲准确性换简洁

## 四、结构示例

> 以下为命理类典籍示例。其他术数参照模式复杂度选择结构，具体章节名称（如"命造验证"→"案例分析"）因书而异。

### 短篇模式（原文简短）

```markdown
## 核心论点

### 论点名称

> 【注家曰】原文

义理解读

> 【另一注家曰】原文

义理解读

（可选）命造验证
```

### 标准模式（理论+命造）

```markdown
## 总论
本篇核心一句话概括

## 理论解析
### 论点一
原文依据 + 义理解读
### 论点二
...

## 命造验证
【命造一】乾造：xxx
格局分析 → 结论

## 关联篇章
（可引用其他篇目的理论关联）
```

### 密集模式（内容丰富）

```markdown
## 总论
本篇核心 + 在体系中的位置

## 理论一
### 原文解读
### 义理展开
### 命造验证

## 理论二
...

## 总结
核心要点 + 进阶路径
```

## 五、禁止行为【红线】

1. **禁止脱离原文解读** — 解读必须有经典原文注家依据
2. **禁止只堆砌原文不解读** — 以典代论违规
3. **禁止自创案例/命造** — 所有案例必须有经典出处
4. **禁止传授网络伪论、民间野诀** — 必须有正统经典依据
5. **禁止流派分歧武断判定** — 分列依据，不偏袒
6. **禁止截取半句引用** — 必须完整
