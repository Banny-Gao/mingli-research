# 前端工程化问题清单

## 一、组件架构

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| 1 | **BookApp.tsx 巨型组件** | P0 | 363 行，6 个 `useEffect`，同时掌控 modal、批注、书签、键盘导航、搜索跳转、目录——应拆为 ModalReader、ActionBar、AnnotationLayer 等子组件 |
| 2 | **Notes.tsx 绕过 hooks 复用** | P1 | 7 个顶层函数（`deleteAnnotation`、`batchDeleteAnnotations`、`deleteBookmark`、`batchDeleteBookmarks`、`loadAllBookmarks`、`loadAllAnnotations`、`exportMarkdown`）直接操作 localStorage，而 `useProgress.ts` 已有封装好的 hooks 模式 |
| 3 | **localStorage key 解析脆弱** | P1 | `split('_', 2)` 在 slug 含下划线时损坏 |

## 二、代码质量

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| 4 | **noUnusedLocals/Parameters 关闭** | P2 | `tsconfig.json` 允许死代码存在，未强制清理 |
| 5 | **无 ESLint** | P2 | 无代码规范检查工具，无法保证代码风格一致 |
| 6 | **TypeScript 类型妥协** | P1 | BookApp.tsx 出现 2 处 `as any` 绕过类型检查 |
| 7 | **SearchBar 模块级可变状态** | P1 | `let searchCache`, `let fuseInstance` 在 React 应用中作为模块级可变变量，应为 ref 或 cache 管理 |
| 8 | **components/index.ts 无用重导出** | P3 | barrel export 仅为 Vite 增量更新而存在，但 Vite 通过 import 路径发现组件，这是多余维护成本 |
| 9 | **SkillGrid.tsx 硬编码映射** | P2 | skill 描述写在 JS 里（`skillDesc: Record<string, { desc, tag }>`），应数据驱动 |
| 10 | **books.ts 类型与数据混合** | P2 | 795 行文件中 TS 类型 + 自动生成的 JSON 数据混在同一文件，应分离为 `.json` |

## 三、CSS 架构

| # | 问题 | 严重度 | 详情 |
|---|------|--------|------|
| 11 | **单体 CSS：1665 行一个文件** | P0 | `src/styles/index.css` 包含所有样式（全局重置、所有页面、所有组件、主题覆写），命名空间无隔离 |
| 12 | **样式与组件不 co-located** | P1 | 无 CSS Modules，无 `.module.css`，每个组件依赖远端单一 CSS 文件中的全局 class，修改一处样式可能影响其他组件 |
| 13 | **Tailwind 已配置但未使用** | P0 | `@theme` 只用来声明变量，6 个组件 + 3 个页面中零处 Tailwind utility class，零处 `@apply` |
| 14 | **CSS @theme 变量未映射到 Tailwind utility 层** | P1 | `--color-gold: #f0c060` 定义了但 `text-gold` / `bg-gold` / `border-gold` 等语义类不在代码中出现 |
| 15 | **inline style 泛滥** | P0 | 75 处 inline style 分布于 9 个文件，其中约 65 处（87%）是静态样式，本应为 class 或 utility |
| 16 | **inline 重复模式严重** | P1 | `display:flex; align-items:center; gap:8` 出现 ~15 次；`{flex:1}` 出现 5 次；`fontSize:12/13` + `color:text-dim` 出现 ~10 次——本可为 `.flex-center` class 或 Tailwind utility |
| 17 | **Notes 用 JS 手写 CSS 类系统** | P2 | `const btnBase: React.CSSProperties` + 每次 spread + 按条件覆盖——本质上在 JS 里手动模拟 CSS 类系统 |
| 18 | **无预处理/嵌套语法** | P2 | 无 Less/Sass，导致 `.prose-interp h1` 重复定义（696-699 行）、`var(--color-gold)` 全文出现 30+ 次 |
| 19 | **light theme 手工同步** | P2 | light 主题覆写（1438-1517）手动匹配每一处 dark 变量，新增组件后需两处维护 |

## 优先级汇总

| 优先级 | 项数 | 标准 | 包含项 |
|--------|------|------|--------|
| **P0** | 4 | 严重影响维护效率或增量开发的架构性问题 | BookApp 巨型组件、单体 CSS、Tailwind 未使用、inline style 泛滥 |
| P1 | 6 | 造成重复工作或潜在 bug | Notes 绕过 hooks、localStorage key 解析、类型妥协、SearchBar 状态管理、样式不 co-located、inline 重复模式、CSS 变量未映射 utility |
| P2 | 8 | 增加维护成本但不阻塞开发 | noUnused 关闭、无 ESLint、硬编码映射、books.ts 混合、无 shadcn、手工 JS 类系统、无预处理、light 主题手工同步 |
| P3 | 2 | 轻微代码卫生问题 | barrel export 无用、重复 h1 定义 |
