## 功能

在收到八字四柱并完成三元分析后，先判定命局的"形象"类别——是两气成象（两种五行均衡）、独象（一种五行独旺）、全象（三种以上五行并存），还是形全形缺（五行当令状态），然后根据形象类别输出对应的用神选取和行运建议。此技能在方局判断和八格判断之前调用，优先判定特殊形象格局。

## 输入

- **四柱八字**：年柱、月柱、日柱、时柱的天干地支
- **五行统计**：天干五行和地支五行的分布计数
- **月令**：月柱地支（用于判定形全形缺）

## 处理逻辑

1. **两气成象判定**
   a. 统计八字中出现的五行种类数
   b. 如果只有2种五行（或某两种五行占绝对主导）：
   - 列出是哪两气（如木火、土金等）
   - 判断是相生关系还是相克关系
   - 检查是否有破局五行混入
   - **相克两气需判定是否"均敌"**（力量均衡）
     c. **依据**：形象篇【任氏曰】"两气双清，非独木火二形也...相克务须均敌，切忌偏重偏轻"

2. **独象判定**
   a. 检查是否为专旺格（一种五行占绝对主导）：
   - 木 → 曲直格（方局全，不杂金）
   - 火 → 炎上格（方局全，不杂水）
   - 土 → 稼穑格（四库全，不杂木）
   - 金 → 从革格（方局全，不杂火）
   - 水 → 润下格（方局全，不杂土）
     b. 检查化神（食伤）是否昌盛
     c. **依据**：形象篇【任氏曰】"权在一人，曲直炎上之类是也。化者，食伤也，局中化神昌旺"

3. **全象判定**
   a. 检查命局是否有3种及以上五行并存
   b. 检查日主是否旺相（全象的基本条件）
   c. 检查财星是否有力
   d. **依据**：形象篇【原注】"三者为一，有伤官而又有财也，主旺喜财旺"

4. **形全形缺判定**
   a. 逐行检查是否当令（得月令生扶为形全，失月令为形缺）
   - 木生寅卯辰月为形全
   - 火生巳午未月为形全
   - 金生申酉戌月为形全
   - 水生亥子丑月为形全
   - 土生辰戌丑未月为形全
     b. 基于形全形缺，决定处理方向：
   - 形全 → 确定用"泄"还是"伤"（食伤泄秀或官杀克制）
   - 形缺 → 确定用"帮"还是"助"（比劫帮扶或印绶生助）
     c. 特殊判定：即使形全也可能不真正"全"，即使形缺也可能不真正"缺"
     d. **依据**：形象篇【原注】"如甲木生于寅、卯、辰月，丙火生于巳、午、未月，皆为形全"

5. **用神与行运建议**
   a. 两气成象 → 顺势引导（用神取秀气所在的五行）
   b. 独象 → 以化神（食伤）为用神，忌官杀破局
   c. 全象 → 以财星为用神，忌比劫夺财
   d. 形全 → "泄伤分用"——判断用食伤泄还是用官杀克
   e. 形缺 → "帮助分用"——判断用比劫帮还是用印绶助

## 输出

```typescript
interface XiangXingReport {
  /** 形象类别判定 */
  category: {
    type: '两气成象' | '独象' | '全象' | '形全形缺' | '普通格局'
    name: string // 如"木火两气成象"、"曲直格"
    description: string
  }
  /** 两气成象详情 */
  liangQi?: {
    elements: [string, string] // 两种五行
    relation: '相生' | '相克'
    balance: '均敌' | '偏重'
    hasPoju: boolean // 是否有破局五行
    pojuElements: string[]
    analysis: string
  }
  /** 独象详情 */
  duXiang?: {
    dominantElement: string
    zhuanWangType: '曲直' | '炎上' | '稼穑' | '从革' | '润下'
    huaShen: string // 化神（食伤）的五行
    huaShenStrength: '昌' | '中' | '弱'
    hasHuaShen: boolean
    analysis: string
  }
  /** 全象详情 */
  quanXiang?: {
    elements: string[] // 所有五行
    caiXing: string // 财星五行
    caiStrength: '旺' | '中' | '弱'
    analysis: string
  }
  /** 形全形缺详情 */
  xingQuan?: Record<
    string,
    {
      isQuan: boolean // 是否形全
      action: '泄' | '伤' | '帮' | '助' | '无'
      actionElement: string // 使用的五行
      reason: string
    }
  >
  /** 综合建议 */
  advice: {
    yongShenElement: string
    preferredLuck: string[]
    avoidLuck: string[]
    warning: string
  }
}
```
